1. ### Используя справочную систему, изучите все опции команд chmod, chown, chgrp, file, umask.

### __chmod__ (Change Mode):
- Команда chmod используется для изменения прав доступа к файлам и директориям.
- Она позволяет устанавливать или снимать разрешения на чтение, запись и выполнение для владельца, группы и остальных пользователей.
- Пример использования: chmod +x filename (добавить право на выполнение файла).

__-R__ (рекурсивное изменение):
Эта опция позволяет изменять права доступа для файлов и каталогов рекурсивно. То есть, если вы примените chmod -R к директории, это затронет все файлы и поддиректории внутри нее.

__-v__ (подробный вывод):
При использовании этой опции команда chmod будет подробно описывать каждое изменение прав доступа. Это полезно, чтобы видеть, какие файлы были изменены.

__-c__ (показать информацию об изменениях):
Эта опция выводит информацию только о файлах, для которых были изменены права доступа. Если вы хотите увидеть только результаты изменений, используйте -c.

### __chown__ (Change Owner):
- Команда chown позволяет изменять владельца файла или директории.
- Пример использования: chown user:group filename (изменить владельца на указанного пользователя и группу).

__-R__ - рекурсивный режим, спускается вниз по иерархии каталогов до файлов и модифицирует их;

__-v__ - выводит подробный отчет о действиях для всех файлов;

__-c__ - аналогичен –v, но отображает только изменения. 

### __chgrp__ (Change Group):
- Команда chgrp изменяет группу, к которой принадлежит файл или директория.
- Пример использования: chgrp newgroup filename (изменить группу на указанную).
- Опции команды chgrp аналогичны опциям команды chown. 

### __file__:
- Команда file используется для определения типа файла.
- Пример использования: file filename (получить информацию о типе файла).

__-b__ (без подробностей):
Запрещает вывод имен и адресов файлов в результате команды.

Пример:
file -b myfile.txt

__-i__ (определение MIME-типа):
Определяет MIME-тип документа на основе его заголовка.

Пример:
file -i image.jpg

__--mime-type и --mime-encoding__:
Определяют конкретный элемент MIME (тип или кодировку).

Пример:
file --mime-type myfile.txt

__-f__ (анализ из файла):
Позволяет анализировать документы, адреса которых указаны в простом текстовом файле.

Пример:
file -f files.txt

### __umask__:
- Команда umask устанавливает маску прав доступа по умолчанию для новых файлов и директорий.
- Пример использования: umask 022 (установить маску, чтобы новые файлы были доступны для чтения и записи владельцем, и только для чтения остальными).

__-S__, маска будет отображаться с использованием символьной нотации. 

Например:
$ umask -S
u=rwx,g=rx,o=rx

## Контрольные вопросы

1. Для каких типов пользователей определяется доступ к файлу?

- Владелец (Owner): Это пользователь, который является владельцем файла или каталога. Владелец имеет полный контроль над файлом и может изменять его права доступа.
- Группа (Group): Группа пользователей, которой принадлежит файл или каталог. Группа может иметь общие права на файлы, и один и тот же пользователь может входить в несколько групп одновременно1.
- Остальные пользователи (Others): Это все остальные пользователи системы. Они не являются владельцами и не входят в группу, связанную с файлом. Права доступа для остальных пользователей также могут быть определены1.

2. На что, кроме права на запись и права на чтение, могут иметь право пользователи?

- Выполнение (Execute) (обозначается буквой “x”): Дает разрешение на выполнение файла или на вход в каталог.

3.Как определить режим доступа к файлу или каталогу?

В Linux режим доступа к файлам определяется с помощью команды chmod. 

__chmod [опции] <права> <файлы>__

Важно понимать основные типы прав доступа, которые назначаются файлам или каталогам:

__Чтение (Read)__ (обозначается буквой “r”): Позволяет просматривать содержимое файла или каталога.

__Запись (Write)__ (обозначается буквой “w”): Позволяет создавать, изменять и удалять файлы внутри каталога, а также изменять содержимое файла.

__Выполнение (Execute)__ (обозначается буквой “x”): Дает разрешение на выполнение файла или на вход в каталог.

4. Опишите действие модификатора доступа Sticky bit на доступ к файлам и каталогам.

__Sticky bit для каталогов__:

- Когда Sticky bit установлен на каталоге, это означает, что только владелец файла может удалить или переименовать файлы внутри этого каталога.

- Другие пользователи, даже если у них есть права на запись в этот каталог, не могут удалять или переименовывать файлы, которые принадлежат другим пользователям.

- Это полезно, например, в общедоступных каталогах, чтобы предотвратить случайное удаление или переименование файлов другими пользователями.

__Sticky bit для файлов__:

- В случае файлов Sticky bit не имеет такого же явного действия, как для каталогов.

- Однако в прошлом Sticky bit использовался для оптимизации работы с исполняемыми файлами в памяти. Например, когда исполняемый файл был запущен, он оставался в оперативной памяти после завершения, чтобы следующие запуски происходили быстрее.

- В современных системах это использование устарело, и Sticky bit для файлов редко используется.

__Установка Sticky bit__:
Чтобы установить Sticky bit на каталоге, используйте команду chmod с опцией +t. 

Например:
chmod +t /путь/к/каталогу

Для удаления Sticky bit используйте опцию -t.

5. Опишите действие модификатора доступа SUID на доступ к файлам и каталогам.

Модификатор доступа __SUID (Setuid)__ — это специальный бит, который может быть установлен на исполняемых файлах в Linux. 

__SUID для исполняемых файлов__:

- Когда бит SUID установлен на исполняемом файле, это означает, что при его запуске процесс получает права владельца файла.

- Например, если обычный пользователь запускает исполняемый файл с установленным SUID, то процесс будет выполняться с правами владельца файла, а не с правами пользователя, который его запустил.

- Это часто используется для программ, которые требуют повышенных привилегий, например, для изменения системных настроек или доступа к файлам, к которым обычные пользователи не имеют доступа.

__Пример использования SUID__:

- Предположим, у нас есть исполняемый файл backup.sh, который регулярно создает резервные копии системных файлов.

- Если установить SUID на этот файл, то при запуске он будет выполняться с правами владельца (например, с правами администратора), что позволит ему создавать резервные копии файлов, к которым обычные пользователи не имеют доступа.

__Установка SUID__:

- Чтобы установить SUID на файле, используйте команду chmod с опцией +s. 

Например:
chmod +s backup.sh

- Для удаления SUID используйте опцию -s.

__Осторожность__:

- Использование SUID может быть опасным, так как это может привести к нежелательным действиям, если файл содержит уязвимости.

- Рекомендуется использовать SUID только для необходимых системных задач и следить за безопасностью таких файлов.

6. Опишите действие модификатора доступа SGID на доступ к файлам и каталогам.

Модификатор доступа __SGID (Setgid)__ — это специальный бит, который может быть установлен на исполняемых файлах в Linux. 

__SGID для исполняемых файлов__:
- Когда бит SGID установлен на исполняемом файле, это означает, что при его запуске процесс получает права группы, которой принадлежит файл.

- Другими словами, использование этого бита позволяет нам запускать программы от имени группы, а не от имени пользователя, который их запустил.

- Это полезно, например, для файлов, которые должны выполняться с определенными привилегиями группы, например, для доступа к общим ресурсам или файлам, к которым группа имеет доступ.

__Пример использования SGID__:

- Предположим, у нас есть каталог shared_data, который используется несколькими пользователями из одной группы.

- Если установить SGID на этот каталог, то все файлы, созданные внутри него, будут автоматически принадлежать той же группе, что и сам каталог.

- Это обеспечивает совместный доступ к файлам между членами группы, даже если они создают новые файлы в этом каталоге1.

__Установка SGID__:

- Чтобы установить SGID на файле, используйте команду chmod с опцией +s. 
  
  Например:
chmod g+s myfile

- Для удаления SGID используйте опцию -s.

__Осторожность__:

- Использование SGID требует внимания к безопасности, так как это может повлиять на доступ к файлам и каталогам. Убедитесь, что группа, на которую установлен SGID, имеет соответствующие права доступа к файлам.
